Browse through the sections below to see how to handle different field mapping strategies:

- **[Basic Mapping](./basic.md)** - Mapping identical fields.
- **[Field Renaming](./rename.md)** - Dealing with mismatched names.
- **[Skip & Default](./skip.md)** - Handling fields that don't exist in the source.
- **[Nested Conversions](./nested.md)** - Automatically converting child structs.
- **[Custom Functions](./custom.md)** - Applying transformation logic.
- **[Combining Attributes](./combined.md)** - Using multiple mapping strategies at once.
